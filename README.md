## 🔍 Core Concepts

### 🎭 Git vs. GitHub: What's the Difference?
*   **Git**: A *local* version control system. It tracks code changes on your computer.
*   **GitHub**: A *remote*, cloud-based hosting service. It lets you share your Git repositories online.

### 🛠️ Key Terminal Commands
Here are the essential commands I am practicing this summer:
*   `git init` — Initializes a *new* local Git repository.
*   `git add .` — Stages all changed files for the next commit.
*   `git commit -m "message"` — Saves a snapshot of the staged changes.
*   `git push origin main` — Sends local commits to the *remote* GitHub server.
*   `git pull` — Fetches and merges changes from GitHub to the *local* machine.

### 💡 Tips for Clean Repos
*   Always write *clear, present-tense* commit messages (e.g., `Fix bug`, not `Fixed bug`).
*   Never upload sensitive data like passwords; use a `.gitignore` file.
*   *Practice makes perfect!* Push code daily to build your GitHub green contribution square streak.
